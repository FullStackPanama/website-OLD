## Introducción

¡Bienvenido a la guía de Full Stack Panamá sobre cómo usar GitHub! Ya seas nuevo en el desarrollo o un programador experimentado, esta guía te ayudará a navegar por GitHub, una herramienta esencial para colaborar en nuestros proyectos.

## ¿Qué es GitHub?

GitHub es una plataforma basada en web utilizada para control de versiones y colaboración, permitiéndote trabajar de manera más efectiva con otros en proyectos.

## Conceptos Básicos de GitHub

### Repositorios

Un repositorio (o "repo") es donde se almacenan los archivos del proyecto. Explora nuestros repositorios para encontrar proyectos que te interesen.

### Commits

Los commits son instantáneas de tu proyecto en un punto específico en el tiempo. Cuando haces cambios en un archivo, haces commit de esos cambios para guardarlos.

### Ramas (Branches)

Las ramas se utilizan para trabajar en nuevas características o correcciones de errores sin afectar el proyecto principal. Puedes crear una nueva rama, hacer cambios y fusionarla de vuelta a la rama principal.

### Forks

Los forks son copias de un repositorio que puedes modificar sin afectar el proyecto original. Puedes hacer fork de un repositorio para trabajar en una característica o corrección de error.

### Issues

Los issues se utilizan para rastrear ideas, mejoras, tareas o errores. Puedes comentar en un issue para expresar interés o hacer preguntas.

### Pull Requests (PRs)

Los PRs te permiten informar a otros sobre los cambios que has enviado a un repositorio. Cuando estés listo para contribuir, envía un PR con tus cambios.

## Haciendo Tu Primera Contribución

<Steps>
  1. **Haz Fork del Repositorio**: Crea una copia del repositorio haciendo fork.
  2. **Clona Tu Fork**: Clona tu fork a tu máquina local con `git clone <url>`.
  3. **Haz Cambios Localmente**: Trabaja en la tarea o característica, asegurándote de seguir las pautas de contribución del proyecto.
  4. **Haz Commit de Tus Cambios**: Usa `git commit` para guardar tus cambios.
  5. **Envía a Tu Fork**: Sube tus cambios a GitHub con `git push`.
  6. **Abre un Pull Request**: Ve al repositorio original en GitHub y haz clic en "Pull Request" para informar al equipo que tienes cambios para fusionar.
</Steps>

## Conclusión

Contribuir a proyectos en GitHub puede parecer intimidante al principio, pero es una forma gratificante de colaborar y aprender. Si tienes alguna pregunta o necesitas ayuda, no dudes en contactar a nuestro equipo o a otros miembros de la comunidad. ¡Feliz codificación!
